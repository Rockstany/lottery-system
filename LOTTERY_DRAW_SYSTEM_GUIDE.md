## What is This System?

Think of this as a **digital spinning wheel** for your church lottery, just like the traditional lottery draws you see on TV or at events. But instead of a physical wheel, we use a computer screen that shows an animated wheel.

**What it does:**
- Shows all lottery ticket numbers on a spinning wheel

- Everyone can watch on a big screen (projector/TV)
- Saves all results automatically
- Cannot be cheated or manipulated

**What it looks like:**
Imagine a big colorful wheel on the screen with all the lottery book numbers written on it. When you click a button, the wheel spins really fast, then slowly stops on one number - that's the winner!

## Security Features Explained Simply

### ğŸ”’ Security Feature #1: The Lockdown

**What it is:**
During the draw, the admin is completely locked out of the system.

**How it works:**
- Computer has a timer (like an alarm clock)
- When draw time comes, timer activates
- Admin pages close automatically
- Admin cannot login even with correct password
- Only Viewer can access

**Real-world example:**
Imagine a bank vault with a time-lock. Even the bank manager cannot open it before the scheduled time. Same concept!

**Why this matters:**
Admin cannot:
- Change lottery numbers
- Remove someone's entry
- Add fake entries
- See results before announcement
- Influence the draw in any way

---

### ğŸ² Security Feature #2: True Randomness

**What it is:**
The computer picks winners using special random number generation (not predictable).

**How it works:**
- Computer uses something called "cryptographic randomness"
- It's like shuffling cards perfectly with your eyes closed
- Uses multiple sources of randomness:
  - Computer's internal clock (down to millionths of a second)
  - Random data from the computer's processor
  - Random data from your mouse movements
  - All mixed together

**Real-world example:**
Imagine throwing 10 dice, shuffling cards, and flipping coins all at the same time. Then mixing all those results together. That's how random it is!

**Why this matters:**
- Nobody can predict the winner
- Nobody can rig the system
- Every eligible book has equal chance
- Mathematically fair

**Fun fact:** This is the same randomness used by banks for security and by casinos for online gambling!

---

### ğŸ” Security Feature #3: Password Change During Draw

**What it is:**
Viewer must change their password immediately after login.

**Why:**
- The admin created the original password
- After you change it, admin doesn't know the new password
- Even if admin writes down the original, it won't work
- Ensures admin cannot login as Viewer during or after draw

**Real-world example:**
Like changing locks on a new house. Previous owner had keys, but after you change locks, their old keys don't work.

**What gets recorded:**

- Can be reviewed by anyone
- Builds trust
